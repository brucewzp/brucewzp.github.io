ubuntu关于nofile的一个bug

2018-05-12 | 运维

发现了一个高版本ubuntu的一个bug，无法修改open file limit。

第一次尝试

敲入ulimit -n 65535
再ulimit -n并没有修改成功

第二次尝试

在/etc/security/limits.conf中添加

再ulimit -n仍没有修改成功

第三次尝试

难道是我打开的方式不对？经过一番搜索，据说要重启一下，于是我怀着忐忑的心情重启了一下，，，，
然而并没有卵用。。。

解决方案

无奈继续在网上搜索，终于找到解决方法了，而且这确实是一个bug
在/etc/systemd/user.conf和/etc/systemd/system.conf 中修改DefaultLimitNOFILE=65535,然后最后重启就可以了！
